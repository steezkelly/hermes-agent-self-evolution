1. **Authenticate with GitHub** – Log in to the Hermes CLI so you can access the repository and PR data:  
   ```bash
   hermes auth login --provider github
   ```

2. **Fetch PR metadata** – Retrieve the PR’s details (base branch, head branch, list of changed files) and store the JSON for reference:  
   ```bash
   hermes pr view <owner>/<repo>#<pr_number> --json base,head,files
   ```

3. **Download the diff** – Pull the full diff of the PR into a local file that you’ll analyze:  
   ```bash
   hermes pr diff <owner>/<repo>#<pr_number> -o pr.diff
   ```

4. **Parse the diff** – Split the diff by file and line numbers. If Hermes provides a built‑in parser, use it:  
   ```bash
   hermes diff parse pr.diff --output parsed_diff.json
   ```
   Otherwise, use a small script (e.g., Python, Node) to read `pr.diff` and emit a list of changed files with line ranges.

5. **Run static analysis for bugs** – For each modified source file, invoke the linter or static‑analysis tool to catch common bug patterns:  
   ```bash
   hermes lint <path_to_file> --language <language>
   ```

6. **Run security scan** – Execute the security scanner on the changed files to detect vulnerabilities such as hard‑coded secrets, injection risks, or insecure dependencies:  
   ```bash
   hermes security scan <path_to_file>
   ```

7. **Run performance analysis** – Use the performance profiling tool to identify potential bottlenecks (e.g., N+1 queries, expensive loops, missing caching):  
   ```bash
   hermes perf analyze <path_to_file>
   ```

8. **Review outputs and categorize findings** – Tally the issues from steps 5‑7, label each as **Bug**, **Security**, or **Performance**, and assign a severity level (Critical, High, Medium, Low).

9. **Post constructive inline comments** – For each actionable issue, create an inline comment on the exact file and line using the Hermes comment command (which calls the GitHub Pull Request Comments API):  
   ```bash
   hermes pr comment <owner>/<repo>#<pr_number> \
     --file <path_in_diff> \
     --line <line_number> \
     --body "<Constructive comment explaining the issue and suggesting a fix>"
   ```
   Repeat this command for every issue you want the author to address.

10. **Submit review decision** – After all comments are posted, submit the overall review status to the PR:  
    ```bash
    hermes pr review <owner>/<repo>#<pr_number> --approve
    # or, if changes are required:
    hermes pr review <owner>/<repo>#<pr_number> --request-changes
    ```
    Include a concise summary of findings in the review body (auto‑generated by Hermes or written manually).

11. **Log the review activity** – Record the outcome for traceability (e.g., in a ticketing system or internal dashboard):  
    ```bash
    hermes log review --pr <owner>/<repo>#<pr_number> --result <approve|request-changes>
    ```
