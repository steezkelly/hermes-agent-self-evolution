# CEO Orchestration — Strategic Planning Layer

## Overview

The CEO Orchestration skill implements the **Executive Agent** role in Steve Kelly's companion system. It provides strategic planning, goal review, and roadmap generation by reading the wiki state and producing structured plans.

## Architecture

```
User Input (strategic language)
    |
    v
[CEO Agent] — reads wiki state, interprets intent
    |
    +---> Reads: index.md, goals/, plans/, SCHEMA.md
    +---> Analyzes: progress, blockers, dependencies
    |
    v
[Plan Generation] — writes to ~/wiki/plans/
    |
    v
[Goal Updates] — updates ~/wiki/goals/ status
    |
    v
[Manager Agent] — distributes sub-tasks (optional)
```

## When to Use

Trigger the CEO agent when the user says:
- "plan", "strategy", "roadmap", "90-day", "quarter"
- "review goals", "priorities", "what should I focus on"
- "create a timeline", "milestone", "objective"
- "mission", "vision", "resource allocation"

## Usage

### Standalone Script

```bash
# Generate a strategic plan
python3 ~/hermes0/scripts/ceo-agent.py "Create a 30-day plan for improving local inference"

# Review current state
python3 ~/hermes0/scripts/ceo-agent.py "Review goals and identify blockers"
```

### Via Hermes delegate_task

```
delegate_task(
    goal="Create a Q3 roadmap based on current progress",
    context="<generated by ceo-agent.py>",
    toolsets=["file"],
    max_iterations=10,
)
```

### Direct Hermes Invocation

```
@ceo-plan Review current goals and suggest next priorities
```

## Agent Roles

| Role | Function | Toolset | Trigger |
|------|----------|---------|---------|
| CEO | Strategic planning, roadmap creation | `file` | "plan", "strategy", "roadmap" |
| Manager | Task decomposition, wiki maintenance | `file` | "organize", "sync", "lint" |
| Researcher | Web research, fact gathering | `web` | "research", "find", "compare" |
| Engineer | Coding, building, debugging | `terminal`, `file` | "build", "code", "fix" |
| Curator | Memory sync, Honcho curation | `web` | "remember", "ingest" |

## Safety Constraints

1. **Single-toolset rule**: CEO gets only `file` (no web+terminal combo)
2. **No recursive delegation**: CEO cannot spawn other agents
3. **Wiki as source of truth**: All output goes to `~/wiki/` before Honcho sync
4. **Schema compliance**: All new pages follow `SCHEMA.md` conventions

## Files

- `~/hermes0/scripts/ceo-agent.py` — State reader + config generator
- `~/hermes0/scripts/spawn-agent.py` — Role-based agent spawner
- `~/hermes0/scripts/intent-router.py` — Natural language intent classifier
- `~/wiki/plans/` — Output directory for strategic plans
- `~/wiki/goals/` — Active goals (updated by CEO)

## State Inspection

The CEO agent automatically reads:

1. **index.md** — Total pages, sections, links
2. **goals/*.md** — Progress checkboxes, status, blockers
3. **plans/*.md** — Existing roadmaps and timelines
4. **SCHEMA.md** — Conventions for new pages

## Plan Format

All plans written by the CEO agent follow this structure:

```markdown
---
type: plan
title: Plan Title
created: YYYY-MM-DD
updated: YYYY-MM-DD
confidence: 0.8
tags: [plan, strategy, quarter]
---

# Plan Title

## Mission Statement
...

## Current State Assessment
...

## Strategic Objectives
...

## Weekly Sprint Cadence
...

## Resource Budget
...

## Success Criteria
...

## Related
- [[goals/build-companion-system|Build Companion System]]
```

## Integration with Other Agents

After the CEO creates a plan:

1. **Manager Agent** breaks it into sub-tasks
2. **Engineer Agent** implements technical items
3. **Researcher Agent** gathers missing information
4. **Curator Agent** syncs verified facts to Honcho

## Troubleshooting

**Issue:** CEO agent times out on large wiki
**Fix:** The `file` toolset with `max_iterations=10` is safe. If the wiki grows very large, increase `max_iterations` or scope the task.

**Issue:** Plans don't follow SCHEMA.md
**Fix:** The CEO context includes SCHEMA.md conventions. If issues persist, run the Manager agent to lint new pages.

**Issue:** Goals not updating
**Fix:** Ensure the goal file has proper frontmatter with `status: active`. The CEO only updates active goals.
